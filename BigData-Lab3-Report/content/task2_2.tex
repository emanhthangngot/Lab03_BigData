\section{Task 2.2: StdDev of Amount by SKU-Month with Percentile Threshold}

\subsection*{Hướng dẫn điền nhanh cho người phụ trách}

\begin{itemize}
    \item File nguồn cần tham chiếu: \texttt{src/Task\_2-2/src/main/scala/Task2\_2.scala}.
    \item Phải nêu rõ \texttt{stddev\_pop} là population standard deviation, degree of freedom bằng 0.

    \item Phải kiểm tra có nhóm \texttt{SKU-month} nào trên 1,000 orders hay không để thảo luận partitioning.

\subsection{Phân tích truy vấn (Query Analysis)}
\textbf{Mục tiêu cần viết.} Trình bày rằng với mỗi \texttt{SKU} trong từng tháng, ta giữ lại các order có số promotion đạt ngưỡng percentile P90 hoặc P80 của chính nhóm \texttt{(SKU, month)}, rồi tính population standard deviation của \texttt{Amount} trên tập order được giữ.

\textbf{Promotion count cần viết.} Ghi cách đếm promotion ids: split cột \texttt{promotion-ids} theo dấu phẩy, trim token, bỏ token rỗng, đếm tất cả promotion bao gồm Amazon-issued promotions. Null hoặc chuỗi rỗng được tính là 0.

\textbf{Percentile threshold cần viết.} Ghi rõ threshold được tính riêng cho từng \texttt{SKU-month}. Một order qualify nếu \texttt{promo\_count >= threshold}. Tránh viết như ngưỡng toàn dataset.

\textbf{Stddev cần viết.} Ghi rõ dùng \texttt{stddev\_pop}, tức chia cho $N$ thay vì $N-1$. Nếu sau khi lọc percentile có ít hơn 2 order thì stddev được set bằng 0 theo yêu cầu đề bài.

\textbf{Hai approach cần viết.}
\begin{itemize}
    \item Approximate: dùng \texttt{percentile\_approx} hoặc \texttt{approx\_percentile}.
    \item Exact: tự tính bằng DataFrame/Dataset operations, ví dụ window \texttt{row\_number}, \texttt{count}, vị trí \texttt{ceil(p * N)}.
\end{itemize}

\textbf{Schema cần điền.} Theo output schema hiện tại:
\begin{itemize}
    \item \texttt{SKU}
    \item \texttt{month}
    \item \texttt{p90\_stddev\_approx}
    \item \texttt{p90\_stddev\_exact}
    \item \texttt{p80\_stddev\_approx}
    \item \texttt{p80\_stddev\_exact}
\end{itemize}

\subsection{Phân rã thành các bước cơ bản (Decomposition)}
Điền từng bước theo pipeline dưới đây. Với mỗi bước, ghi rõ DataFrame tương ứng trong code nếu có.

\begin{enumerate}
    \item \textbf{Load CSV và chọn cột.} Ghi các cột cần dùng: \texttt{SKU}, \texttt{Date}, \texttt{promotion-ids}, \texttt{Amount}. Nêu cách cast \texttt{Amount} sang Double.
    \item \textbf{Parse month.} Ghi cách parse nhiều format ngày và tạo cột \texttt{month = yyyy-MM}.
    \item \textbf{Tính \texttt{promo\_count}.} Ghi cách split, filter token rỗng và count; null/rỗng thành 0.
    \item \textbf{Tạo base DataFrame.} Ghi base gồm \texttt{SKU}, \texttt{month}, \texttt{promo\_count}, \texttt{amount\_val}, và được cache vì dùng cho cả hai approach.
    \item \textbf{Approximate thresholds.} Ghi groupBy \texttt{SKU, month}, tính P90/P80 bằng built-in, join threshold lại base, lọc order qualify và tính stddev.
    \item \textbf{Exact thresholds.} Ghi window partition by \texttt{SKU, month}, order by \texttt{promo\_count}, gán \texttt{row\_number}, tính \texttt{total\_n}, lấy vị trí \texttt{ceil(p * N)} và threshold tương ứng.
    \item \textbf{Tính stddev và xử lý nhóm nhỏ.} Ghi groupBy \texttt{SKU, month}, count qualifying orders, dùng \texttt{stddev\_pop}; nếu count nhỏ hơn 2 thì trả 0.
    \item \textbf{Ghép kết quả cuối.} Ghi join các DataFrame kết quả P90/P80 approximate/exact về cùng schema.
    \item \textbf{Ghi single Parquet.} Ghi cách coalesce/write temp directory/rename file nếu code làm như vậy.
\end{enumerate}

\subsection{Lý do phân rã}
Viết đoạn giải thích vì sao phải tính \texttt{promo\_count} trước: percentile không tính trên Amount mà tính trên số lượng promotions của order. Sau khi có threshold mới lọc order và tính stddev trên Amount.

Viết đoạn giải thích vì sao threshold phải join lại base DataFrame: threshold là thuộc tính cấp nhóm, còn quyết định order qualify là thuộc tính từng dòng order.

Viết đoạn giải thích vì sao cần hai approach: approximate nhanh và tận dụng built-in aggregation; exact tốn sort/window nhưng cho threshold tái lập chính xác theo quy ước nearest-rank.
