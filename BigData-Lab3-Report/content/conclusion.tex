\subsection{Bài học rút ra}

\begin{enumerate}
    \item \textbf{Sliding window $\leftrightarrow$ emission pattern:} Thay vì quét ngược lại quá khứ cho mỗi ngày tính toán, phương pháp ``đẩy xuôi'' (emit đóng góp cho các ngày tương lai) biến sliding window thành bài toán gom nhóm key-value, hoàn toàn phù hợp với MapReduce.

    \item \textbf{Median đòi hỏi full sort:} Không thể dùng Combiner hay reduceByKey cho median vì median không có tính associative. Bài học: xác định rõ tính chất toán học của aggregation trước khi chọn API.

    \item \textbf{Broadcast join cho bảng nhỏ:} Trong Task 2-1, bảng state average ($\sim$36 dòng) nhỏ đủ để broadcast, tránh shuffle trên bảng orders lớn. Nguyên tắc: luôn kiểm tra kích thước bảng join để chọn strategy phù hợp.

    \item \textbf{Approximate vs exact percentile:} Task 2-2 cho thấy \texttt{approx\_percentile} (Greenwald-Khanna) nhanh hơn nhưng có thể cho threshold khác exact (đặc biệt trên nhóm nhỏ hoặc phân bố skewed). Exact percentile qua \texttt{row\_number} + \texttt{ceil(p $\times$ N)} đảm bảo chính xác nhưng tốn shuffle + sort.

    \item \textbf{Chuẩn hóa dữ liệu:} Dữ liệu thực tế có nhiều vấn đề: khoảng trắng thừa, chữ hoa/thường không nhất quán, null values, format ngày đa dạng. Việc \texttt{trim} + \texttt{lower/upper} ngay đầu pipeline là bắt buộc để tránh sai lệch kết quả.
\end{enumerate}

\subsection{Khó khăn và hướng cải thiện}

\textbf{Khó khăn gặp phải:}
\begin{itemize}
    \item Parse ngày tháng: dataset chứa nhiều format date khác nhau, cần cascade qua nhiều pattern.
    \item Promotion-ids: chuỗi comma-separated với khoảng trắng không đều, trailing comma tạo token rỗng.
    \item Cấu hình Spark trên Linux native: cần cố định đúng bản Spark/Scala/Java và đặt \texttt{SPARK\_LOCAL\_IP}, \texttt{SPARK\_LOCAL\_HOSTNAME} để đảm bảo driver bind ổn định ở local mode.
\end{itemize}

\textbf{Hướng cải thiện:}
\begin{itemize}
    \item Bổ sung unit test tự động cho từng bước transformation, đối chiếu output với expected values.
    \item Chuẩn hóa schema input bằng một bước tiền xử lý chung cho cả bốn task.
    \item Tối ưu số partition dựa trên kích thước dữ liệu thực tế thay vì dùng giá trị mặc định.
\end{itemize}
